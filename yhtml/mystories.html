    One of options I considered was to design an abstraction of test execution interface

    If the followings are test execution interface
    1) run unit
    2) run smoke
    3) run regression
    4) run performance

    1) mojito unit test run commandline
    2) maven/testng based test run commandline
    3) jmeter test run commandline
